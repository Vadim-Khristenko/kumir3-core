# Система пользовательских библиотек Kumir 3

## Обзор

Kumir 3 поддерживает создание пользовательских библиотек на языке Kumir. Библиотеки могут быть:
- Простыми .kum файлами с функциями
- Полноценными проектами с kumir.toml конфигурацией

## Быстрый старт

### 1. Создание простой библиотеки

Создайте файл `моя_библиотека.kum`:

```kumir
алг цел удвоить(цел x)
| Удваивает число
нач
    знач := x * 2
кон

алг цел утроить(цел x)
| Утраивает число
нач
    знач := x * 3
кон
```

### 2. Использование библиотеки

```kumir
использовать "моя_библиотека.kum"

алг Главный
нач
    вывод удвоить(5)  | 10
    вывод утроить(3)  | 9
кон
```

## Продвинутое использование

### Структура проекта библиотеки

```
моя_библиотека/
├── kumir.toml          # Конфигурация
├── src/
│   └── lib.kum         # Исходный код
├── README.md           # Документация
└── examples/           # Примеры использования
```

### Конфигурация kumir.toml

```toml
[package]
name = "моя_библиотека"
version = "1.0.0"
authors = ["Ваше Имя <email@example.com>"]
description = "Описание библиотеки"
license = "MIT"
kumir_version = "3.0.0"
edition = "2024"
type = "library"
main = "src/lib.kum"

# Экспортируемые функции
exports = [
    "удвоить",
    "утроить"
]

# Зависимости от других библиотек
[dependencies]
время = "^2.0"

# Настройки сборки
[build]
main_file = "src/lib.kum"
output_dir = "./build"
optimization_level = 2
debug_info = false
bounds_check = true
strict_mode = true

# Метаданные библиотеки
[library]
category = "математика"
tags = ["math", "utils"]
min_kumir_version = "3.0.0"
stable = true
```

### Импорт библиотеки с конфигурацией

```kumir
использовать "путь/к/моя_библиотека"

алг Главный
нач
    вывод удвоить(10)
кон
```

## Примеры

См. директорию `examples/`:
- `test_user_library/` - полноценная библиотека с kumir.toml
- `test_direct_file.kum` - пример импорта .kum файла
- `library_kumir.toml` - подробный пример конфигурации

## Технические детали

### Загрузка библиотек

Система автоматически определяет тип библиотеки:
1. Если путь указывает на .kum файл → загружается как простая библиотека
2. Если путь указывает на директорию с kumir.toml → загружается как проект
3. Если путь не содержит расширения → ищется встроенная библиотека

### Кэширование

Загруженные библиотеки кэшируются для повышения производительности. Повторный импорт той же библиотеки использует кэшированную версию.

### Метаданные

При загрузке из kumir.toml извлекаются:
- Имя и версия библиотеки
- Автор и описание
- Зависимости
- Настройки сборки
- Экспортируемые функции

### Интеграция с интерпретатором

Функции пользовательских библиотек регистрируются в LibraryManager и доступны для вызова так же, как встроенные функции.

## Ограничения

- Пользовательские библиотеки не имеют нативных обработчиков (handler = None)
- Функции выполняются через интерпретатор
- Рекурсивные зависимости не поддерживаются

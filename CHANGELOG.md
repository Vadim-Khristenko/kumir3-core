# Changelog - Kumir 3 Core

## [0.2.0-alpha.1] - 2026-06-19

### Добавлено

#### Стандарт языка KITE
- Серия документов **KITE 0–11** (`arch/kite/`) — аналог Python PEP: обзор языка, система типов,
  асинхронность, окружения, библиотеки, режим совместимости с КуМир 2, STDIO, гайдлайны,
  циклы разработки, движок типов и объектная модель. Тёмная HTML-тема, живой поиск, переключатель тем.

#### Движок системы типов
- Расширяемый движок `shared/src/typesys/` — единый источник правды: правила совместимости,
  унификация/общий тип, план приведения (`coercion`), значения по умолчанию, типизация операций.
  Используется интерпретатором (`Environment::type_system`) и компилятором (`TypeChecker`).
- Легаси `TypeKind::is_assignable_from`/`common_type` делегируют движку (дубля логики нет).

#### Объектная модель (ООП)
- Стабильная идентичность объектов (`type_id` через `TypeRegistry`), диспетчеризация методов по
  иерархии наследования, полиморфизм с переопределением, вызов `предок.метод()`.
- Проверки: запрет инстанцирования абстрактных классов, запрет переопределения `финал`,
  обязательная реализация абстрактных методов; диспетчеризация методов из `реализация … для …`;
  контроль видимости полей (`открытый`/`закрытый`/`защищённый`).

#### Окружения
- Лексическая видимость переменных (без «утечки» через стек вызовов), блочные области (`push_scope`).

#### Новые типы
- **`диапазон`** (Range): литералы `1..10` и `1..=10`, перебор в `нц для x в …` (плюс перебор
  массивов/множеств/кортежей/строк через новый `LoopForEach`).
- **`байты`** (Bytes): `байты("…")`, `строка_из_байт`, `длина`.

#### Библиотеки
- Корректные пользовательские библиотеки: прямой `.kum`, соседний файл (база — каталог скрипта),
  и библиотека-проект с `kumir.toml` (функции вызываемы).

#### Инфраструктура
- GitHub Actions: сборка под Windows/macOS/Linux + релиз по тегу `vX.Y.Z`.
- Git-хуки (`.githooks/`): Conventional Commits, авто-вырезание ИИ-упоминаний, требование CHANGELOG при смене версии.
- Политика линтов `[workspace.lints]` для зелёного `clippy -D warnings` на стадии альфы.

### Изменено
- Версия проекта поднята до `0.2.0-alpha.1`.
- Кодовая база отформатирована (`cargo fmt`) и очищена под `clippy`.

  - Поддержка путей к .kum файлам и директориям

  - `examples/test_user_library/` - полноценная тестовая библиотека с kumir.toml

  - `test_example_library_parses` - проверка парсинга примера библиотеки
  - `test_user_library_loader` - проверка загрузки из файла

#### Система импорта библиотек с версиями
- Поддержка версионных спецификаций в импортах: `использовать время@^2.0`
- `find_library_with_version()` в `shared/src/libraries/registry.rs`
- `import_versioned()` в `interpreter/src/interpreter/library_bridge.rs`
- Обработчик импортов в `interpreter/src/interpreter/executor.rs`
- Поддержка файловых импортов (.kum файлы)
- Алиасы: `использовать время как т`
- Выборочный импорт: `использовать время { now_ms, sleep }`

#### Структура компилятора
- **CLI компилятора** (`compiler/src/main.rs`)
  - Поддержка целей: exe, wasm, ir, rust
  - Уровни оптимизации: -O 0-3
  - Режим отладки: --debug
  - Проверка синтаксиса: --check
  - Вывод промежуточных форматов: --emit-ir, --emit-rust

- **Главный модуль** (`compiler/src/compiler.rs`)
  - Координация процесса компиляции
  - Интеграция с лексером и парсером
  - Управление оптимизациями

- **AST → IR конвертер** (`compiler/src/ast_to_ir.rs`)
  - Преобразование алгоритмов в IR функции
  - Обработка инструкций: присваивание, вывод, условия, циклы
  - Обработка выражений: литералы, переменные, операции, вызовы
  - Конвертация типов и операций

- **Backend инфраструктура** (`compiler/src/backend/`)
  - Trait `Backend` для бэкендов компиляции
  - `InterpreterBackend` - выполнение IR напрямую
  - `RustBackend` - генерация Rust кода и компиляция через rustc
    - Поддержка всех базовых IR инструкций
    - Генерация кода для LoadConst (числа, строки, bool, char)
    - Обработка унарных и бинарных операций
    - Встроенные функции: print, input, assert
    - Правильная генерация логических операций (возврат 0/1)

- **Оптимизатор IR** (`compiler/src/optimizer.rs`)
  - Уровень 1: Свёртка констант (constant folding)
    - Вычисление арифметических операций на этапе компиляции
    - Вычисление логических операций
    - Вычисление операций сравнения
  - Уровень 2: Удаление мёртвого кода (dead code elimination)
    - Удаление неиспользуемых переменных
    - Анализ использования через все инструкции
  - Уровень 3: Агрессивные оптимизации (заготовка для будущего)
  - Режим отладки для отслеживания оптимизаций

#### Архитектура компилятора
```
Source Code → Lexer → Parser → AST → IR Builder → Backend → Output
     ↓           ↓        ↓       ↓        ↓           ↓         ↓
  (string)   (tokens) (Program) (AST)  (IrModule)  (codegen)  (binary)
```

#### Поддерживаемые конструкции в AST → IR
**Инструкции (Stmt):**
- Присваивание переменных (Assignment, ArrayAssignment)
- Объявление переменных (VarDecl, AutoVarDecl)
- Вывод (Output, OutputFormatted)
- Ввод (Input)
- Условные операторы (If-then-else)
- Циклы: while (LoopWhile), for (LoopFor), do-while (LoopDoWhile), infinite (LoopInfinite)
- Возврат значений (Return, ReturnValue, ResultAssign)
- Утверждения (Assert)
- Выражения как инструкции (ExprStmt)
- Блоки инструкций (Block)
- Пустая инструкция (Nop)

**Выражения (Expr):**
- Литералы (числа, строки, символы, логические)
- Переменные (Variable)
- Бинарные операции (+, -, *, /, %, ^, ==, !=, <, <=, >, >=, и, или)
- Унарные операции (-, не)
- Вызовы функций (Call)
- Доступ к элементам массива (ArrayAccess)
- Условные выражения (IfExpr)
- Доступ к полям (FieldAccess)
- Вызовы методов (MethodCall)
- Приведение типов (Cast)
- None/null значения

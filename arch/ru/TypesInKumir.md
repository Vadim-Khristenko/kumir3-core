## 🛠 Полная классификация типов и ключевых слов

Кумир 3 — это мультипарадигменный язык. Ниже приведен подробный обзор системы, разделенной на логические блоки.

### 1. Фундаментальные типы (Primitive Types)
В дополнение к классике (`цел`, `вещ`, `лог`, `сим`, `лит`), Кумир 3 вводит:
- **авто** (`AutoType`) — для тех, кто ценит лаконичность. Компилятор сам поймет, что вы имели в виду.
- **пустота** (`NoneType`) — стандартный `void` или `()`, когда возвращать нечего.
- **необязательно** (`OptionalType`) — безопасная работа с отсутствующими значениями (аналог `Option<T>` в Rust).

### 2. Управление памятью и Указатели (The Low-Level Power)
Кумир 3 дает вам полный контроль (и полную ответственность):
- **указатель** — тип данных для хранения адреса.
- **новый** / **удалить** — ручное управление кучей.
- **ссылка** (`&`) и **разыменовать** (`^`) — для безопасного и прямого доступа к данным.
- **заимствовать** — временный доступ к данным без передачи владения.
- **клонировать** — глубокое копирование для создания независимых копий.
- **копировать** — эффективное поверхностное копирование простых типов.

### 3. Объектно-Ориентированная Модель (ООП)
Теперь Кумир — это полноценный объектный язык:
- **класс** и **структура** — для организации данных.
- **интерфейс** и **свойство** (`trait`) — для полиморфизма.
- **реализация** (`impl`) — блок, где вдыхается жизнь в типы (методы).
- **предок** (`super`) и **я** / **себя** (`self`) — для навигации по иерархии объектов.
- **открытый** / **закрытый** — для тех, кто любит инкапсуляцию.

### 4. Функциональная магия и Асинхронность
- **лямбда** — анонимные функции для обработки данных на лету.
- **совпадение** (`match`) — мощнейший инструмент проверки шаблонов. Больше никаких бесконечных `если-то-иначе`.
- **асинх** и **ждать** — современный стандарт написания неблокирующего кода.

### 5. Модульная система
Кумир 3 — это командный игрок:
- **модуль** — логическое объединение кода.
- **подключить** — импорт внешних файлов и библиотек.
- **экспорт** — управление видимостью ваших функций для внешнего мира.

### 6. Новые расширенные токены (Kumir 3+)
Чтобы описывать современные практики, добавлены новые лексемы:
- **измен** (`Mut`) — явная мутабельность полей и переменных; упрощает анализ владения и предупреждает скрытые изменения.
- **перемещение** (`Move`) — семантика перемещения для владения ресурсами; сигнализирует о передаче без копирования.
- **заимствовать** (`Borrow`) — заимствование ссылки без передачи владения; позволяет временный доступ к данным.
- **клонировать** (`Clone`) — глубокое копирование значения; создаёт независимую копию с новыми ресурсами.
- **копировать** (`Copy`) — поверхностное копирование для копируемых типов; эффективно для простых данных.
- **где** (`Where`) — секция ограничений для обобщённых алгоритмов и типов; фиксирует требования к параметрам.
- **типалиас** (`TypeAlias`) — создание читаемых псевдонимов типов без новых сущностей.
- **когда** (`When`) — охранные выражения внутри `match`/`при`; разделяют шаблон и дополнительное условие.
- **запустить** (`Spawn`) — запуск конкурентной/асинхронной задачи как «легкого» потока.
- **уступить** (`Yield`) — кооперативная передача управления: генераторы и корутины отдают исполнение, сохраняя состояние.
- **отложить** (`Defer`) — откладывает выполнение блока до выхода из области видимости; удобен для гарантированного освобождения ресурсов.

## 🦀 Мост в мир Rust (Native Inserts)

Одной из самых безумных и крутых фич является прямая интеграция с Rust. Если вам не хватает производительности или нужны возможности системных библиотек, вы можете использовать:

```kumir
РастВставкаНЦ
    // Здесь живет чистый, быстрый и безопасный Rust-код
    let x: i32 = 42;
    println!("Hello from the metal!");
РастВставкаКЦ

```

Или краткое ключевое слово **ржавчина** для быстрых манипуляций. Поддерживаются как однострочные вставки (`RustInline`), так и многострочные блоки (`RustCode`).

## ⚡ Операторы и Синтаксический сахар

Мы добавили операторы, которые делают код чище:

* `**` — возведение в степень.
* `:=`, `+=`, `-=` и т.д. — современные операторы присваивания.
* `|>` — конвейер (Pipe), чтобы строить цепочки вызовов как в Elixir или F#.
* `?` — оператор раннего возврата ошибки (как в Rust).

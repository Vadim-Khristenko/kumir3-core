1. Философия типов

Kumir 3 использует градуальную типизацию: можно писать с авто и любой, а можно — со строгими типами и проверкой совместимости. В ядре статический дескриптор TypeKind зеркалит рантайм-значение Value, поэтому любое значение умеет сообщить свой тип, а типы являются значениями первого класса (рефлексия через тип).

Движок реализован «Умный детектор» вместо большого match уже есть: расширяемый движок правил (shared/src/typesys) — совместимость, унификация, приведение, типизация операций. Подробности — в KITE 10. Этот документ описывает сами типы; движок описывает отношения между ними.

2. Числовые типы

11. Правила совместимости

  • Тождество: одинаковые типы совместимы.
  • любой/авто: принимают любое значение.
  • пусто → T?: null допустим там, где ждут Option.
  • Числовое расширение: меньший числовой тип присваивается большему совместимой категории (напр. цел_16 → цел_64; целое → вещественное большего размера).
  • Общий тип: для бинарных операций выбирается тип большего размера/точности.
Без скрытых сужений Сужающие преобразования (например цел_64 → цел_8) не происходят автоматически — нужно явное приведение.
